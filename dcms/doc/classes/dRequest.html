dRequest - обработка переменных переданных пользователем. Вся обработка запроса происходит в конструктер. Для доступа к переменным есть набор функций выполняющих проверку переменной и приведение ее к нужному типу ( форме ).
  • int - преобразование переменной к типу int, переменная может быть отрицательной;
  • intID - неотрицательный, не нулевой тип. То есть переменная будет приведена к типу int, и будет выполнена проверка что ее значение >=1;
  • textID - текстовый идентификатор. Результат строка, в которой есть только символы a-zA-Z0-9-_., все остальное будет заменено на _;
  • textLine - строка текста. Все спец символы будут экранированы. Подходит для заголовков тем и прочего в одну строку.
  • url - валидный url адрес;
  • bool - булевая переменная;
  • flag - проверяет указана ли переменная в запросе, при этом игнорируется ее значение. Возвращает true|false;
  • raw - чистые данные из запроса. После экранирования.
  • float - число с плавающей точкой;
  • select - выбор переменной из нескольких вариантов. $mode = D::$req->select('mode', array('html','bbcode'));

Маппинг переменных запроса на объект

Когда необходимо изменить множество переменных в одном объекте, можно использовать мапинг. Пример приведен ниже: D::$req->map($object1,array('title'=>'textLine', 'content'=>'html','group_id'=>'intID'));
